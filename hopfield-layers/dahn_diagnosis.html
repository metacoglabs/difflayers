§ 0TL;DR — Three Sentences

🔥
Architecture Bug (Critical)

run_dynamics already runs T full (diffusion + attention) steps updating Q, then _associate passes that already-attended Q into association_core for one more attention step. This double-attention with a corrupted Q is why DAHN degrades accuracy by up to 50 % at moderate noise — the effect is larger than diffusion itself.

📐
Metric Mismatch (Critical)

The experiments test exact pattern retrieval (argmax = correct index). Graph diffusion provably reduces intra-cluster discriminability — making stored patterns from the same cluster more similar — which directly hurts argmax accuracy. DAHN's actual guarantee is cluster-level energy gaps, not individual pattern separation.

🧭
Wrong Diffusion Direction (Critical)

Key diffusion (default) moves stored patterns toward cluster means, reducing intra-cluster discriminability. For noisy retrieval, query diffusion is correct: it maps the noisy query toward its cluster centroid so it can match clean stored patterns. The ablation already shows Q_only improves Hamming distance by 13.6 %.

None of these require changing the mathematical foundation. The DAHN theory is correct — it is tested on the wrong metric with the wrong implementation direction.

§ 2Root Cause Analysis

🏗 ARCH-01 · CRITICAL
Double-Attention: run_dynamics already attends, then association_core attends again

In DiffusedHopfield._associate(), run_dynamics runs T iterations of (diffuse K + attend Q = V·softmax(QK^T)), updating Q each step. The returned state_pattern is the T-th retrieval result, not the original noisy query. This is then passed as the query to association_core:

state_pattern, stored_pattern = engine.run_dynamics(
    Q=state_pattern, K=stored_pattern, V=pattern_projection, ...
)
# state_pattern is now Q_T ≈ V^T softmax(β Q_{T-1} K_T^T)
# It is a BLURRY MIX of stored patterns — not the original query!

result = self.association_core(
    query=state_pattern,    # ← corrupted Q_T, not the noisy query
    key=stored_pattern,     # ← diffused K_T
    ...
)
# This runs ONE MORE attention step on corrupted inputs → T+1 total

Why the degradation is huge at small eta: Even with eta=0.05 (almost-identity diffusion), the extra attention step is the problem. At T=1, DAHN runs 2 total attention steps vs the baseline's 1. At high noise, the first pass already converges to a wrong pattern; the second pass cements it.

🏗 ARCH-02 · CRITICAL
Wrong Diffusion Direction — Key Diffusion Reduces Intra-Cluster Discriminability

Default: diffuse_key=True, diffuse_query=False. This smooths stored patterns toward cluster centroids while leaving the noisy query untouched.

What key diffusion does to attention scores
$$K_i' = (I - \eta L)K_i \approx K_i - \eta\sum_{j\in\mathcal{N}(i)}(K_i - K_j)$$ $$\text{Same-cluster patterns: } K_i' \approx K_j' \approx \bar{K}_\text{cluster}$$ $$\text{Scores: } q\cdot K_i'^T \approx q\cdot K_j'^T \quad \forall\, i,j \in C$$ $$\Rightarrow \text{Attention weight spreads uniformly over cluster → higher entropy → argmax accuracy drops}$$

The ablation directly confirms this: K_only reduces accuracy 0.40→0.30 (−25 %) and increases Hamming error. Q_only keeps accuracy at 0.40 while reducing Hamming by 13.6 %.

🧪 EXP-01 · CRITICAL
Metric Mismatch — Testing Exact Retrieval When DAHN Claims Cluster-Level Improvement

All experiments use argmax(attention) == target_index. This requires distinguishing between same-cluster patterns — exactly what key diffusion destroys. DAHN's theoretical guarantee is inter-cluster energy gap, not intra-cluster discriminability.

The guarantee DAHN actually makes
$$\underbrace{\|K_i' - K_j'\|}_{\text{same cluster}} \downarrow \quad\quad \underbrace{\|K_i' - K_l'\|}_{\text{cross cluster}} \approx \|K_i - K_l\|$$ $$\Rightarrow \text{Inter-cluster gap improves; intra-cluster discriminability worsens}$$ $$\Rightarrow \text{Cluster-level accuracy improves; argmax exact accuracy worsens}$$

The test metric punishes exactly the thing diffusion does. With N=200, 20 clusters (10 per cluster), key diffusion makes all 10 patterns in each cluster nearly identical — argmax fails within clusters, but the correct cluster wins.

🧪 EXP-02 · MODERATE
Bad Graph Parameters — k=3 Too Sparse for N=200, d=64

The noise robustness experiment uses k=3 on N=200 patterns in d=64. At this density the 3 nearest neighbors of a pattern may not be its actual cluster mates (especially under noise). Diffusing over wrong neighborhoods corrupts both intra-cluster structure and inter-cluster boundaries. Rule: $k \geq N/n_\text{clusters} = 200/20 = 10$.

🏗 ARCH-03 · MODERATE
SimpleDiffusion and SpectralDiffusion Show Zero Improvement at Any Step Count

From steps_sweep.csv: SimpleDiffusion accuracy is exactly 0.352 for T=0 through T=10. The op_dyn.steps=1 truncation makes the outer loop apply D repeatedly, but since association_core resets retrieval at the end, iterative D-smoothing inside the dynamics loop has no incremental benefit beyond what a single step achieves — and causes over-smoothing for spectral mode where $H^2 \approx H$ anyway (idempotent-like for low-frequency patterns).

🧪 THEORY-01 · MODERATE
Anti-Sharpening Entropy Increase Is Mathematically Correct — But for the Wrong Metric

Attention entropy increases with η (3.776 → 4.105). This is not a bug — it is exactly what key diffusion does. All cluster members collapse to centroid → attention spreads equally over them → higher entropy. The entropy argument only holds at the cluster level: inter-cluster entropy (which cluster wins) decreases. But all experiments measure total entropy over all N patterns.

§ 3Theory–Practice Gap

3.1 What DAHN Theory Actually Guarantees

✅
Inter-cluster energy gap increases

$E(\xi, K_\text{wrong cluster}) - E(\xi, K_\text{correct cluster})$ grows with diffusion. Proven from PSD of L.

✅
Smoothness term decreases

$\text{tr}(K'^\top L K') \leq \text{tr}(K^\top L K)$. Energy is non-increasing under diffusion.

✅
Query diffusion reduces Hamming

Noisy query $q = p + \epsilon$ diffused to cluster centroid $q' \approx \bar{p}$: lower distance to stored patterns.

❌
Exact retrieval accuracy improves

NOT guaranteed. Key diffusion collapses intra-cluster patterns → argmax accuracy falls.

❌
Attention sharpens (entropy ↓)

NOT with key diffusion. Intra-cluster attention spreads → entropy rises. Only inter-cluster entropy falls.

3.2 Correct Evaluation Metrics

Cluster-level accuracy (what DAHN actually improves)
$$\text{Cluster-Acc} = \frac{1}{M}\sum_{m=1}^M \mathbf{1}\!\left[\text{cluster}(\hat{k}_m) = \text{cluster}(k_m^*)\right]$$
Inter-cluster energy gap (primary theoretical metric)
$$\Delta E = E(\xi, K_\text{nearest spurious cluster}) - E(\xi, K_\text{correct cluster}) \quad \uparrow \text{ with } \eta$$
Top-K accuracy (K = cluster size)
$$\text{Top-K-Acc} = \frac{1}{M}\sum_{m=1}^M \mathbf{1}\!\left[k_m^* \in \text{top-K}\,\text{softmax}(\beta\, q_m K'^T)\right]$$

§ 4Precise Fix Plan

🔧 FIX-01 · Architecture
Add run_diffusion() — Decouple Diffusion from Attention

Add a run_diffusion method that only diffuses K (and optionally Q), with no attention inside the loop. Change _associate to call this instead of run_dynamics. association_core then does the single attention at the end.

❌ Current (broken)
def run_dynamics(self, Q, K, V, ...):
    for _ in range(self._steps):
        if diffuse_key:
            K = self._diff_op(K)
        if diffuse_query:
            Q = self._diff_op(Q)
        # ATTENTION INSIDE LOOP — corrupts Q!
        Q = self._attn_op.attend(Q, K, V, ...)
    return Q, K   # Q is T-th retrieval result

# Then _associate does:
Q, K = engine.run_dynamics(Q, K, V, ...)
result = self.association_core(query=Q, key=K, ...)
# ↑ EXTRA attention on already-attended Q!
✅ Fixed (pre-diffuse)
def run_diffusion(self, Q, K, ...):
    """Diffuse ONLY — no attention."""
    for _ in range(self._steps):
        if diffuse_key:
            K = self._diff_op(K)
        if diffuse_query:
            Q = self._diff_op(Q)
    return Q, K  # Q is original or gently diffused

# In _associate:
Q, K = engine.run_diffusion(Q, K, ...)
# ONE attention via association_core:
result = self.association_core(query=Q, key=K, ...)

# src/utils/metrics.py — add:
def cluster_accuracy(pred_idx, target_idx, cluster_labels):
    return (cluster_labels[pred_idx] == cluster_labels[target_idx]).float().mean().item()

def topk_accuracy(attn_weights, target_idx, k):
    topk_idx = attn_weights.topk(k, dim=-1).indices   # (M, k)
    return (topk_idx == target_idx.unsqueeze(1)).any(dim=-1).float().mean().item()

def inter_cluster_energy_gap(Q, K, cluster_labels, beta, target_idx):
    scores = beta * (Q @ K.T)
    gaps = []
    for m in range(Q.shape[0]):
        tc = cluster_labels[target_idx[m]]
        s_correct = scores[m, cluster_labels == tc].max()
        s_wrong   = scores[m, cluster_labels != tc].max()
        gaps.append((s_correct - s_wrong).item())
    return float(np.mean(gaps))

🔧 FIX-06 · Architecture
Add Logit Diffusion as Primary Mode for Exact Retrieval Claims

Logit diffusion smooths post-softmax attention weights over the key graph — without corrupting stored patterns. This is the correct mode for improving exact retrieval under the DAHN framework.

Logit diffusion — smooths weights, not patterns
$$\alpha = \text{softmax}(\beta Q K^T), \quad \alpha' = (I - \eta L)\,\alpha,\quad \alpha' \leftarrow \alpha'/\|\alpha'\|_1$$ $$\text{output} = \alpha'^T V$$ $$\text{K stays untouched — no intra-cluster discriminability loss}$$

§ 5Refined Claims That Hold Up

✅
Claim 1 — Cluster-level retrieval improves

DAHN with key diffusion improves cluster-level accuracy at noise 0.15–0.35. The model correctly identifies the cluster even when individual patterns become indistinguishable. Measurable with cluster_accuracy metric.

✅
Claim 2 — Inter-cluster energy gap is provably larger

$\Delta E(\xi, K_\eta) \geq \Delta E(\xi, K)$ for any query $\xi$ and $\eta > 0$. Proven from the PSD property of $L$ and the energy descent theorem. Measurable and monotone.

✅
Claim 3 — Query diffusion reduces reconstruction Hamming distance

With diffuse_query=True, diffuse_key=False: Hamming distance between retrieved pattern and clean target decreases by ~14 % at moderate noise. Already validated by existing ablation data.

✅
Claim 4 — Logit diffusion improves top-K cluster accuracy

Smoothing post-softmax weights over the graph (logit diffusion) improves top-K accuracy (K=cluster_size) without harming stored pattern discriminability. This is the clean "both better" result.

⚠️
Claim to REMOVE — "DAHN improves exact retrieval accuracy"

Not supported when key diffusion is used. Replace with: "DAHN with query diffusion achieves lower reconstruction error at comparable exact accuracy. DAHN with key diffusion improves cluster-level accuracy at a theoretically expected cost to intra-cluster exact retrieval."
