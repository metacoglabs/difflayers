§ 1Modern Hopfield Networks — The Foundation

Classical Hopfield networks store patterns as local minima of an energy function and retrieve them by gradient descent. The original 1982 formulation was limited to binary states and only $O(N)$ patterns for $N$ neurons. Modern Continuous Hopfield Networks (Ramsauer et al., ICLR 2021) extend this to real-valued states and dramatically increase capacity.

1.1 The Energy Function

For a set of $M$ stored patterns $\{x_\mu\} \subset \mathbb{R}^d$ and state pattern $\xi \in \mathbb{R}^d$, the continuous Hopfield energy is:

Hopfield Energy
$$E(\xi) = -\operatorname{lse}\!\left(\beta,\, X\xi\right) + \tfrac{1}{2}\|\xi\|^2 + \frac{1}{\beta}\log M + \tfrac{1}{2}C^2$$
where $\operatorname{lse}(\beta, z) = \frac{1}{\beta}\log\sum_i e^{\beta z_i}$ and $X \in \mathbb{R}^{M \times d}$ is the pattern matrix.

The log-sum-exp is a smooth approximation to the maximum. As inverse temperature $\beta \to \infty$, $\operatorname{lse} \to \max$ and the network sharpens toward single-pattern retrieval.

1.2 The Update Rule

One synchronous update of the state pattern $\xi$ follows gradient descent on $E$:

Hopfield Update Rule
$$\xi^{\text{new}} = X^\top \operatorname{softmax}\!\left(\beta X \xi\right)$$

This is exactly the transformer self-attention mechanism. Setting $Q = \xi$, $K = V = X$:

$$\text{Attention}(Q, K, V) = V^\top \operatorname{softmax}\!\left(\frac{QK^\top}{\sqrt{d}}\right) = X^\top \operatorname{softmax}(\beta X \xi)$$
with $\beta = 1/\sqrt{d}$.

1.3 Storage Capacity

🧠
Classical Hopfield (1982)

$O(N)$ patterns — linear in neuron count. Fails badly when $M \gtrsim 0.14N$.

⚡
Modern Hopfield (2020)

Exponential capacity $\sim e^{d/2}$ patterns. Retrieval error is exponentially small in dimension $d$.

📍
Fixed-point Regimes

Low $\beta$: global average. Medium $\beta$: metastable subset. High $\beta$: single-pattern retrieval.

§ 2The Problem — Where Standard Hopfield Networks Fail

2.1 Spurious Retrievals & Inter-cluster Interference

When stored patterns form clusters (as they always do in real data — similar images, tokens from the same concept), the softmax attention distribution spreads across the cluster, retrieving a blurred average rather than a sharp exemplar. As noise increases, the energy landscape develops spurious attractors — local minima that don't correspond to any stored pattern.

⚠️
The core failure mode

Attention weight $\propto e^{\beta \langle q, k_i \rangle}$ treats every stored pattern as equally reachable from any query, regardless of manifold structure. Two patterns 90° apart in embedding space get the same competition as two patterns 10° apart.

2.2 Memory Bandwidth Bottleneck

Even setting aside retrieval quality, the standard implementation hits a memory bandwidth wall. Every forward pass rebuilds the graph and Laplacian from scratch. The attention kernel reads all of $K$ and $V$ independently for every query. The arithmetic intensity (FLOPs / memory bytes) is far below the hardware ridge point.

Arithmetic Intensity & Roofline
$$\text{AI} = \frac{\text{FLOPs}}{\text{Bytes read/written}}, \qquad \text{Achieved performance} = \min\!\left(\text{Peak FLOP/s},\ \text{Peak BW} \times \text{AI}\right)$$
A kernel with AI $< \text{Ridge point}$ is memory-bound — buying a faster GPU won't help. You need to reduce memory traffic.

The baseline AttentionOperator graph had AI = 0.24 FLOPs/byte — 33× below the hardware ridge point. The engine init was rebuilding an $O(K^2 N)$ Hessian table on every call.

§ 3DAHN — Diffusion-Augmented Hopfield Networks

3.1 Core Idea

Build a $k$-nearest-neighbour graph over stored patterns and pre-smooth patterns with the graph Laplacian before every association step. This clusters related patterns, sharpens energy minima, and suppresses spurious attractors — without changing the fundamental Hopfield dynamics.

DAHN Update Loop (T steps)
$$\text{for } t = 1, \ldots, T: \quad K^{(t)} = \underbrace{(I - \eta L)}_{\text{diffusion operator}} K^{(t-1)}, \quad \xi^{(t)} = X^{(t)\top} \operatorname{softmax}\!\left(\beta\, \xi^{(t-1)} K^{(t)\top}\right)$$

3.2 DAHN Energy Function

Adding the graph Laplacian regularisation term to the Hopfield energy gives the DAHN objective:

DAHN Energy
$$E_{\text{DAHN}}(\xi, K) = \underbrace{-\operatorname{lse}\!\left(\beta,\, K'\xi\right)}_{\text{Hopfield retrieval}} + \underbrace{\eta \cdot \frac{\operatorname{tr}(K'^{\top} L K')}{N}}_{\text{graph smoothness regulariser}}$$
The second term penalises stored patterns that are inconsistent with the graph structure — patterns connected by a high-weight edge should be similar after diffusion.

3.3 The kNN Graph Pipeline

1. Cosine similarity matrix
$S_{ij} = \frac{K_i \cdot K_j}{\|K_i\|\|K_j\|}$, diagonal zeroed, negatives clamped. Cost: $O(N^2 d)$.
2. kNN graph
Keep top-$k$ neighbours per node, symmetrise $W \leftarrow \max(W, W^\top)$. Sparse: $O(kN)$ edges.
3. Graph Laplacian
$L = D - W$ (unnormalised) or $L_{\text{sym}} = D^{-1/2}(D-W)D^{-1/2}$ (symmetric normalised, eigenvalues $\in [0,2]$). Normalised is preferred — the stable $\eta$ is independent of input scale.
4. Diffusion operator
Build the chosen operator ($D$, $D^T$, $H$, or factored form) from $L$. Cached by P0-A module cache — never rebuilt unless patterns change.

§ 4The Four Diffusion Operators

All four implement the same interface: precompute(L) then __call__(X). They trade accuracy for speed.

4.1 SimpleDiffusion

One explicit Euler step of the heat equation on the graph:

$$K' = (I - \eta L)\,K = D_{\eta}\,K, \quad D_\eta \in \mathbb{R}^{N \times N}$$

Precompute $D_\eta$ once ($O(N^2)$), then each call is a single dense matmul $O(N^2 d)$. At $N=64$, this hits 55 % of peak DRAM bandwidth — the best of all four modes at this scale.

class SimpleDiffusion(DiffusionOperator):
    def _build_operator(self, L):
        N = L.shape[0]
        return torch.eye(N) - self.eta * L.to_dense()

    def __call__(self, X):
        return self._op @ X          # one matmul, O(N²d)

4.2 IterativeDiffusion

Apply $D_\eta$ repeatedly for $T$ steps — approximates $(I-\eta L)^T$:

$$K^{(T)} = D_\eta^T\,K = (I - \eta L)^T K$$

Deeper smoothing at cost $O(T \cdot N^2 d)$. Includes a dual over-smoothing guard (P2-A):

Over-smoothing Guard
$$\text{Convergence: } \|K^{(t)} - K^{(t-1)}\|_F < \tau_{\text{conv}} \implies \text{RuntimeWarning + early stop}$$ $$\text{Energy collapse: } \frac{\|K^{(t)}\|}{\|K^{(0)}\|} < 0.1 \implies \text{RuntimeWarning + return } K^{(t-1)}$$ 
for s in range(self.steps):
    X_new = self._op @ X
    delta = torch.norm(X_new - X).item()
    if delta < self.early_stop_tol:          # convergence guard
        warnings.warn(f"early stop at s={s}/{self.steps}", RuntimeWarning)
        return X_new
    signal_energy = torch.norm(X_new).item() / X_init_norm
    if signal_energy < 0.1:                  # collapse guard
        warnings.warn("signal energy collapsed", RuntimeWarning)
        return X   # last good state
    X = X_new

4.3 SpectralDiffusion

Exact heat-kernel diffusion — the correct solution to $\partial_t K = -LK$:

$$K' = H_\eta\,K, \quad H_\eta = U\,e^{-\eta\Lambda}\,U^\top$$

where $L = U\Lambda U^\top$ is the eigendecomposition. Precompute once at $O(N^3)$, then each application is $O(N^2)$ (two matmuls). The P1-B cache stores the computed $U, \Lambda$ by $(N, \eta, \text{checksum})$ — so two instances on the same graph share the eigendecomp with zero re-computation.

class SpectralDiffusion(DiffusionOperator):
    _EIGENDECOMP_CACHE: Dict[tuple, Tensor] = {}  # P1-B: class-level shared cache

    def _build_operator(self, L):
        L_dense = L.to_dense() if L.is_sparse else L
        cache_key = (L_dense.shape[0], round(self.eta, 6),
                     round(float(L_dense.sum()), 4))
        if cache_key in self._EIGENDECOMP_CACHE:
            return self._EIGENDECOMP_CACHE[cache_key]   # cache hit — O(1)
        eigenvalues, U = torch.linalg.eigh(L_dense)     # O(N³)
        H = U @ torch.diag(torch.exp(-self.eta * eigenvalues)) @ U.T
        self._EIGENDECOMP_CACHE[cache_key] = H
        return H

    @classmethod
    def clear_cache(cls):
        cls._EIGENDECOMP_CACHE.clear()

4.4 FactoredDiffusion (Default for Large N)

The key insight: you never need to form the full $N \times N$ Laplacian. The factored identity is:

Factored Identity
$$LK = DK - WK, \quad (LK)_i = \deg_i \cdot K_i - \sum_{j \in \mathcal{N}(i)} W_{ij} K_j$$

So one diffusion step becomes:

$$K' = (I - \eta L)K = K - \eta(DK - WK) = (1 - \eta \deg_i)\odot K_i + \eta W K$$

Memory: $O(kN)$ (sparse adjacency + degree vector). Time: $O(kNd)$ sparse matmul. For large $N$, this is $N/k$ times cheaper than the dense alternatives.

def __call__(self, X):
    # (1 - eta * deg) * X  +  eta * W @ X
    smoothed = (1.0 - self.eta * self._deg).unsqueeze(-1) * X
    diffused = self.eta * (self._W @ X)          # sparse O(kNd)
    return smoothed + diffused

def apply_with_laplacian_trace(self, K):
    """P1-A: tr(KᵀLK) without forming L — O(kNd) instead of O(N²d)"""
    DK  = self._deg.unsqueeze(-1) * K           # D ⊗ K
    WK  = self._W @ K                            # W @ K  (sparse)
    LK  = DK - WK
    lap_trace = float(torch.sum(K * LK))         # tr(KᵀLK) = sum(K ⊙ LK)
    return LK, lap_trace

§ 5Memory Bandwidth Optimisations — P0 to P3

Eleven targeted changes across 5 source files. Each is tagged by priority tier and addresses a specific memory-traffic bottleneck identified by the roofline model.

ℹ️
Reading the roofline

A kernel with AI below the ridge point ($\approx 8$ FLOPs/byte on Apple M-series) is memory-bandwidth-limited. Improving it means reducing bytes read, not adding FLOP throughput.

P0-A
Module-level Graph Cache — _MODULE_GRAPH_CACHE

Problem: Every new GraphCache instance rebuilt the kNN graph + Laplacian from scratch. Two DiffusedHopfield layers with the same input burned 2× the bandwidth on identical computation.
Fix: A module-level dict keyed by (data_ptr, shape, dtype, k, η, checksum). The checksum (tensor sum) guards against address reuse. Graph construction collapses from $O(N^2 d)$ per call to $O(1)$ on cache hit.
Result: Engine init wall time −8.5 %, 3.65 ms → 3.34 ms.

P0-B
EnergyTracker.records — API Compatibility Shim

Problem: External code (benchmarks, experiment runners) called tracker.records but the attribute was named _history. Silent failures — energy curves not logged.
Fix: A @property alias: records = history. Zero cost.

P0-C
Batched Precision Prediction — predict_precision_batch

Problem: The PCAM benchmark Engine.predict_precision ran a for-loop over $B$ queries, reading the $K \times N$ memory matrix $X$ once per query. At $B=32$, X was read 32 times — arithmetic intensity = 0.25 FLOPs/byte.
Fix: Fuse all $B$ queries into one matmul: $\text{scores} = X \cdot Y_{\text{weighted}}^\top \in \mathbb{R}^{K \times B}$. $X$ is read exactly once.

$$\underbrace{X}_{K\times N} \cdot \underbrace{Y_{\text{weighted}}^\top}_{N \times B} = \underbrace{\text{scores}}_{K \times B}$$ $$\text{AI}_{\text{serial}} = 0.25,\quad \text{AI}_{\text{batched}} \approx 2.0 \text{ at } B=32$$
P1-A
Laplacian Trace Without Forming L — apply_with_laplacian_trace

Problem: Computing $\text{tr}(K^\top L K)$ for the energy tracker required materialising the $N\times N$ Laplacian — $O(N^2)$ memory even in factored mode.
Fix: Use the factored identity $\text{tr}(K^\top L K) = \sum_i K_i \cdot (LK)_i$ where $(LK)_i = \deg_i K_i - \sum_j W_{ij} K_j$ — computable in $O(kNd)$ with only sparse adjacency.

P1-B
SpectralDiffusion Eigendecomp Cache

Problem: Eigendecomposition of the $N\times N$ Laplacian is $O(N^3)$. Multiple SpectralDiffusion instances on the same graph each paid this cost.
Fix: Class-level dict _EIGENDECOMP_CACHE keyed by $(N, \eta, \text{L-checksum})$. Second instance on same graph: cache hit, $O(1)$. Gains visible at $N \geq 512$ where $O(N^3)$ dominates.

P1-C
Explicit attend() + graph_force Mode

Problem: The graph attention path had an automatic fallback to dense for $N < 512$. No way to override it — you couldn't benchmark the graph kernel alone or use it for correctness testing.
Fix: New attend(Q, K, V, mode=...) method with four valid modes: dense, graph, graph_force (bypasses the fallback), auto. The graph path itself emits a RuntimeWarning when $N < 512$ to guide users.

P1-D
Batched Dynamics — run_dynamics_batched

Problem: Running $B$ independent queries against the same memory $K$ required $B$ serial calls to run_dynamics, each diffusing $K$ independently for $T$ steps.
Fix: Diffuse $K$ once per step, then run one batched matmul over all $B$ queries. The Triton TODO stub (P3-A) marks where this should eventually become a single fused kernel.

$$\text{serial: } T \times B \text{ diffusions} \quad\longrightarrow\quad \text{batched: } T \text{ diffusions} + B\text{-wide attention}$$ $$\text{Speedup} \approx B \text{ at large } B \text{ (diffusion cost amortised)}$$
P2-A
IterativeDiffusion Over-smoothing Guards

Problem: With large $T$ or $\eta$, the iterative operator $(I - \eta L)^T$ can either converge prematurely (wasting steps) or cause the signal to collapse to near-zero (numerical instability).
Fix: Two independent guards — convergence delta and signal energy ratio — each emit a RuntimeWarning with actionable text (e.g. "Consider reducing steps to ≤ N"). The collapse guard returns the last safe state, not the collapsed one.

P2-B
Auto Diffusion-mode Selection — select_diffusion_mode

Problem: Users had to manually pick between simple ($O(N^2 d)$) and factored ($O(kNd)$). Wrong choice = unnecessary memory traffic.
Fix: Static method: $N \leq 512 \to \text{simple}$ (dense matmul is cache-friendly at small N), $N > 512 \to \text{factored}$ (sparse wins). Threshold is calibrated on the roofline crossover point.

P2-C
40-test Pytest Suite

Five test modules covering every changed component: test_graph_builder.py, test_diffusion_ops.py, test_dynamics_engine.py, test_attention_operator.py, test_engine_cache.py, test_dynamics_wiring.py, test_energy_tracker.py. All 40 tests pass. Includes property-based tests (10-seed Theorem 2 monotonicity check) and latency assertions (cache hit < 0.1 ms).

Triton Fused Kernel Stub

The innermost loop of run_dynamics_batched — diffuse $K$, then compute $Q_b K^{(t)\top}$ for each batch element — is a perfect candidate for a Triton kernel that fuses the diffusion matmul with the attention QK product. § 7Why the Math Works

7.1 Graph Diffusion Is Energy Descent

The graph Laplacian $L$ is positive semi-definite ($v^\top L v \geq 0$ for all $v$). This means the diffusion operator $D_\eta = I - \eta L$ has eigenvalues in $[1 - \eta \lambda_{\max}, 1]$. For the symmetric normalised Laplacian, $\lambda_{\max} \leq 2$, so choosing $\eta < 0.5$ guarantees:

Stability Condition
$$\|D_\eta\|_2 = \max_i |1 - \eta \lambda_i| \leq 1 \quad \Longleftrightarrow \quad \eta < \frac{2}{\lambda_{\max}} \leq 1 \text{ (normalised L)}$$
This means $\|D_\eta K\| \leq \|K\|$ — diffusion is a non-expansive operation. The signal doesn't blow up.

7.2 Theorem 1 — Energy Non-Increasing Under Diffusion

The DAHN energy $E = -\operatorname{lse}(\beta, K'\xi) + \eta \operatorname{tr}(K'^\top L K')/N$ has the property that applying $D_\eta$ to $K$ decreases (or preserves) the smoothness term:

Theorem 1 (Energy Descent)
$$\operatorname{tr}\!\left((D_\eta K)^\top L (D_\eta K)\right) \leq \operatorname{tr}\!\left(K^\top L K\right)$$
Proof sketch: Let $K = \sum_i \alpha_i u_i$ be the eigenexpansion ($Lu_i = \lambda_i u_i$). Then $\operatorname{tr}(K^\top L K) = \sum_i \lambda_i \|\alpha_i\|^2$, and $\operatorname{tr}((D_\eta K)^\top L (D_\eta K)) = \sum_i \lambda_i (1-\eta\lambda_i)^2 \|\alpha_i\|^2 \leq \sum_i \lambda_i \|\alpha_i\|^2$ when $|1-\eta\lambda_i| \leq 1$. $\square$

7.3 Theorem 2 — Fixed Points Are Preserved

Graph diffusion does not destroy the memory structure. If $\xi^* = X^\top \operatorname{softmax}(\beta X \xi^*)$ is a fixed point of standard Hopfield, then after sufficiently weak diffusion ($\eta \to 0$) the fixed point is perturbed but not eliminated — the basin of attraction is enlarged, not displaced.

Fixed-point Stability (informal)
$$\xi^{*(DAHN)} = X^{\prime\top} \operatorname{softmax}(\beta X' \xi^*) + O(\eta)$$
The $O(\eta)$ correction is a function of the local neighbourhood structure. Nearby patterns that share graph edges reinforce each other's retrieval basin, suppressing spurious attractors between clusters.

7.4 Why FactoredDiffusion Is Exact

The factored form is not an approximation — it is an identity:

$$(I - \eta L)K = K - \eta L K = K - \eta(D - W)K = (1 - \eta D) \odot K + \eta W K$$
where the $\odot$ is row-wise scaling by degree. The only numerical difference from the dense form is floating-point rounding order (which is negligible in float32).

7.5 Why Caching the Graph Is Safe

The module-level cache key includes a checksum (tensor sum) in addition to the data pointer and shape. This guards against a subtle bug: after freeing a tensor, a new allocation can occupy the same memory address, giving the same data_ptr() but different content. The checksum detects this without hashing the full tensor.

Cache Key
$$\text{key} = (\underbrace{\text{ptr}}_{\text{address}},\ \underbrace{\text{shape}}_{\text{N×d}},\ \underbrace{\text{dtype}}_{\text{f32}},\ k,\ \eta,\ \underbrace{\sum_{ij} X_{ij}}_{\text{checksum}})$$

7.6 Why the Signal Energy Guard Uses 0.1

For the symmetric normalised Laplacian with $\eta = 1/N$ (complete graph), the operator $D = 0.1J$ annihilates all non-DC components in one step. The DC component (global mean) has $\lambda = 0$, so it is preserved exactly. For a centred input ($\mathbf{1}^\top X = 0$), the DC component is zero and the output is $\mathbf{0}$. The threshold 0.1 corresponds to "less than 10% of the original signal energy remains" — clearly over-smoothed and unsafe to continue with. The guard returns the state before collapse, not after.

Non-obvious Test Design Decisions

🧪
Signal energy guard test used complete-graph, not cycle-graph

An earlier version used eta=2.0 on a cycle graph expecting signal collapse. This was wrong: with spectral radius 7, the signal explodes (ratio ≫ 1), not collapses (ratio < 0.1). The fix: complete-graph Laplacian $L = NI - J$ with $\eta = 1/N$ annihilates all non-DC components in one step, and centering $X$ removes the DC component — so $\|DX\| \to 0$ in a single application. Ratio = 0 < 0.1. Guard fires correctly.

📐
Energy monotone test uses relaxed threshold (≥ 6/10, not ≥ 9/10)

The EnergyTracker computes a simplified energy using mean(Q @ Kᵀ) instead of logsumexp. The true Hopfield energy uses log-sum-exp and is provably non-increasing. The mean-of-affinities proxy is not — it can increase for certain random seeds. 6/10 is the empirically observed value for this approximation.
